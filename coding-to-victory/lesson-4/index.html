Lesson Challenge

I learned there are a lot of commands in git that can be very confusing at first. Once you use them over and over, and see how other people use them, they become more familiar and easier to use.

  • branching: create an offshoot of a working directory
  • pull: to allow others to make changes or review work
  • issues: track and manage bugs, features and changes
  • code review: review and improve changes by contributors
  • workflow: ensure you're doing work where your supposed to. and keep it synced
